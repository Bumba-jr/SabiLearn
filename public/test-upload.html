Upload Endpoint Test

Purpose: Test the upload endpoint with different file sizes to identify the body size limit issue.

Instructions:
  1. Open browser console (F12) to see detailed logs
  2. Check terminal for server-side logs
  3. Test with progressively larger files

Test 1: Route Accessibility (GET)

Tests if the route file is correctly configured and accessible.

Test 2: Tiny File Upload (< 1KB)

Tests if small files can be uploaded successfully.

Test 3: Small File Upload (1-2MB)

Tests if files under typical limits work.

Test 4: Medium File Upload (5MB)

Tests if files at our validation limit work.

Test 5: Custom File Upload

Upload your own file to test.
